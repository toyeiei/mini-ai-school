# Pipes และการรวมคำสั่ง

Pipe `|` ช่วยให้คุณเชื่อมคำสั่งเข้าด้วยกัน—เหมือนสายพานการผลิต

## Pipe คืออะไร?

Pipe `|` นำผลลัพธ์ของคำสั่งหนึ่งไปส่งให้อีกคำสั่ง

```bash
command1 | command2
```

นี่รัน `คำสั่ง1` แล้วส่งผลลัพธ์ไปให้ `คำสั่ง2`

## ตัวอย่าง: ผลลัพธ์ยาวทำให้เป็นประโยชน์

สมมติคำสั่งแสดง 1000 บรรทัด คุณอ่านไม่หมด:

```bash
ls -la
```

แต่ด้วย `head` คุณเห็นแค่ 10 บรรทัดแรก:

```bash
ls -la | head
```

หรือด้วย `tail` 10 บรรทัดสุดท้าย:

```bash
ls -la | tail
```

## ตัวอย่าง: ค้นหาในผลลัพธ์

หา "error" ในผลลัพธ์ของคำสั่ง:

```bash
cat log.txt | grep "error"
```

เนื้อหาไฟล์ไหลไปยัง grep ซึ่งกรองหา "error"

## ตัวอย่าง: นับผลลัพธ์

นับว่ามีกี่ไฟล์ที่มี "readme" ในชื่อ:

```bash
ls | grep readme | wc -l
```

สามคำสั่งเชื่อมกัน!

## ทำไมสำคัญ

Pipes ช่วยให้คุณสร้างเครื่องมือซับซ้อนจากส่วนเล็กๆ แทนที่จะมีคำสั่งเดียวทำทุกอย่าง คุณรวมคำสั่งเล็กๆ ที่แต่ละอันทำสิ่งเดียวได้ดี

นี่เรียกว่า "Unix philosophy"—เครื่องมือเล็ก รวมกันได้ยืดหยุ่น

## ฝึกฝน

ลองรวมคำสั่งเหล่านี้:

```
# Find files modified recently
ls -lt | head

# Search case-insensitive
cat notes.txt | grep -i "meeting"

# Count lines in all .txt files
cat *.txt | wc -l
```
