# What is Version Control

Version control tracks changes to your files over time. It lets you recall specific versions later.

## The Problem

- "final.txt", "final_v2.txt", "final_FINAL.txt"...
- Accidentally deleted important work?
- Want to see who changed what?

Version control solves all of this.

## What Git Does

- **Saves snapshots** of your project
- **Tracks changes** line by line
- **Enables branching** for experiments
- **Allows collaboration** without chaos

## Key Terms

| Term | Meaning |
|------|---------|
| **Repository** | A project folder tracked by Git |
| **Commit** | A saved snapshot |
| **Branch** | A parallel version of code |
| **Merge** | Combining branches together |

## Git vs GitHub

- **Git** - The tool that tracks changes locally
- **GitHub** - A website that hosts Git repositories online

You use Git commands on your computer. GitHub stores your code in the cloud.

## Why It Matters

1. **Undo mistakes** - Revert to any previous version
2. **Track changes** - See who changed what when
3. **Work together** - Multiple people on same project
4. **Backup** - Code lives in the cloud

## Summary

- Version control tracks file changes over time
- Git saves snapshots called "commits"
- GitHub hosts repositories online
- Essential for any development project
