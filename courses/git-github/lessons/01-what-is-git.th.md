# Version Control คืออะไร

Version control ติดตามการเปลี่ยนแปลงของไฟล์ของคุณตามเวลา ทำให้คุณสามารถเรียกดูเวอร์ชันเฉพาะได้ในภายหลัง

## ปัญหา

- "final.txt", "final_v2.txt", "final_FINAL.txt"...
- ลบงานสำคัญโดยไม่ตั้งใจ?
- อยากรู้ว่าใครเปลี่ยนอะไร?

Version control แก้ไขทั้งหมดนี้

## Git ทำอะไรได้บ้าง

- **บันทึก snapshot** ของโปรเจกต์ของคุณ
- **ติดตามการเปลี่ยนแปลง** ทีละบรรทัด
- **รองรับ branching** สำหรับการทดลอง
- **อนุญาตให้ทำงานร่วมกัน** ได้อย่างไม่ยุ่งเหยิง

## คำศัพท์สำคัญ

| คำศัพท์ | ความหมาย |
|------|---------|
| **Repository** | โฟลเดอร์โปรเจกต์ที่ Git ติดตาม |
| **Commit** | snapshot ที่บันทึกไว้ |
| **Branch** | เวอร์ชันคู่ขนานของโค้ด |
| **Merge** | รวม branch เข้าด้วยกัน |

## Git vs GitHub

- **Git** - เครื่องมือที่ติดตามการเปลี่ยนแปลงในเครื่อง
- **GitHub** - เว็บไซต์ที่เก็บ Git repository ไว้ออนไลน์

คุณใช้คำสั่ง Git บนคอมพิวเตอร์ของคุณ GitHub เก็บโค้ดของคุณบนคลาวด์

## ทำไมมันถึงสำคัญ

1. **ยกเลิกความผิดพลาด** - กลับไปเวอร์ชันก่อนหน้าได้
2. **ติดตามการเปลี่ยนแปลง** - ดูว่าใครเปลี่ยนอะไรเมื่อไหร่
3. **ทำงานร่วมกัน** - หลายคนในโปรเจกต์เดียวกัน
4. **สำรองข้อมูล** - โค้ดอยู่บนคลาวด์

## สรุป

- Version control ติดตามการเปลี่ยนแปลงของไฟล์ตามเวลา
- Git บันทึก snapshot เรียกว่า "commit"
- GitHub เก็บ repository ไว้ออนไลน์
- จำเป็นสำหรับทุกโปรเจกต์การพัฒนา
