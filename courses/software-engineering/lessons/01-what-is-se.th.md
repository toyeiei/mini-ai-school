# Software Engineering คืออะไร

Software engineering คือการปฏิบัติงานในการออกแบบ สร้าง และดูแลระบบซอฟต์แวร์ โดยผสมผสานความรู้ด้าน computer science หลักการวิศวกรรม และการจัดการโครงการ เพื่อสร้างซอฟต์แวร์ที่เชื่อถือได้ ขยายได้ และดูแลรักษาได้ง่าย

## Software Development Life Cycle

ทุกโครงการซอฟต์แวร์จะผ่านวงจรดังนี้:

1. **Requirements** - ซอฟต์แวร์ควรทำอะไร?
2. **Design** - เราจะสร้างอย่างไร?
3. **Implementation** - เขียนโค้ด
4. **Testing** - ตรวจสอบว่าทำงานได้ถูกต้อง
5. **Deployment** - เปิดให้ผู้ใช้งาน
6. **Maintenance** - แก้ไขปัญหาและเพิ่มฟีเจอร์

## ความแตกต่างจากการเขียนโค้ด

Programming เน้นการเขียนโค้ด ในขณะที่ Software engineering ครอบคลุมวงจรทั้งหมด:

- **Coding** คือการเขียนคำสั่งให้คอมพิวเตอร์ทำงาน
- **Software Engineering** คือการใช้แนวทางเชิงระบบเพื่อสร้างซอฟต์แวร์ที่แก้ปัญหาจริง

## ทักษะหลักของ Software Engineer

- **Problem decomposition** - แตกปัญหาที่ซับซ้อนออกเป็นส่วนเล็กๆ
- **Communication** - เข้าใจ requirements และอธิบายวิธีแก้ปัญหา
- **Debugging** - ค้นหาและแก้ไขปัญหาอย่างเป็นระบบ
- **Testing** - มั่นใจว่าโค้ดทำงานถูกต้อง
- **Version control** - จัดการการเปลี่ยนแปลงโค้ดตามเวลา
- **Reading documentation** - เข้าใจวิธีการทำงานของ tools และ frameworks

## สิ่งที่คุณจะเรียนรู้ในคอร์สนี้

คอร์สนี้ครอบคลุมพื้นฐานที่ Software engineer ทุกคนต้องมี:

- Problem-solving และ algorithm design
- Data structures และการเลือกใช้
- Design patterns สำหรับปัญหาที่พบบ่อย
- Version control ด้วย Git
- Testing strategies
- Debugging techniques
- Code quality และ maintainability
- พื้นฐาน DevOps

Software engineering คืองานฝีมือ เมื่อคุณฝึกฝนพื้นฐานเหล่านี้ คุณจะพัฒนาอินทรีย์ในการสร้างซอฟต์แวร์ที่ยั่งยืน
